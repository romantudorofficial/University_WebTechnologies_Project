1. Introduction

1.1. Purpose

The purpose of this document is to present a detailed description of the whole web application "Manners Matter!". It will explain the purpose and features of the entire software application, its interfaces, what the software will do and the constraints under which it must operate. This document is intended for users of the software and also potential developers. This is version 1.0.

1.2. Document Conventions

This document was created based on the IEEE template for System Requirements Specification documents, using the English language, as well as the UTF-8 character encoding standard.

1.3. Intended Audience and Reading Suggestions

This document is written towards typical users, such as individuals who are interested in the online etiquette, as well as towards the software developers working on this project.

1.4. Product Scope

"Manners Matter!" is a web application that anyone can use to learn about the online etiquette rules.

1.5. References

You can find the codebase of the web application here and the tasks backlog, used for efficiently splitting the work between the team members, here.

2. Overall Description

2.1 Product Perspective

"Manners Matter!" was developed for everyone who is interested in the online etiquette rules.

2.2 Product Functions

  • Creating Individual or Administrator Accounts
  • Going Through Lessons
  • Solving Questions About the Learned Content
  • Keeping a Scoreboard of the Best Users
  • Suggesting Tips Based on Profile Information

2.3 User Classes and Characteristics

  • Typical Users, such as individuals.
  • Developers who are working on the project.

2.4 Operating Environment

Any Operating System, any web browser, desktop or mobile as well, which supports HTML5.

2.5 Design and Implementation Constraints

The web application is built using HTML and CSS, so it is very easy to modify it.

2.6 User Documentation

The interface is intuitive enough. No need of further documentation to use the software product.

2.7 Assumptions and Dependencies

The web application may look slightly different depending on the web browser used, as well as the device used.
